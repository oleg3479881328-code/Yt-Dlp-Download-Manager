## Local Project Knowledge

### Pattern — Personal local utility can remain compact

Для личного Windows-инструмента не требуется преждевременная архитектурная переработка только из-за наличия двух интерфейсов или двух способов запуска. Если текущая боль не подтверждена, достаточно явно зафиксировать компромисс и проверять затронутый исполнительный контур при изменениях.

### Pattern — Validation before refactor

Если code review (проверка кода) выявил вероятный дефект в ключевом пользовательском сценарии, следующий шаг должен быть коротким validation run (проверочным запуском), а не немедленной переделкой архитектуры или добавлением функций.

### Risk Pattern — Post-processed media path

При использовании `yt-dlp` с FFmpeg post-processing (постобработкой), путь, полученный до завершения конвертации, может не совпадать с путём итогового файла. Для функций «открыть результат» и «скачать результат» нужно подтверждать именно final output path (путь итогового файла).
