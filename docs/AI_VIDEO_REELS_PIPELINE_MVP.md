# AI Video Reels Pipeline MVP

## Owner Direction

The project focus is now a local video pipeline for generating short Reels/Shorts-style clips from longer YouTube videos.

This is **not** a Chrome extension task and not a generic downloader task.

The intended workflow is:

```text
YouTube video
→ get transcript / subtitles with timestamps
→ AI analyzes transcript
→ AI selects semantic episodes
→ user chooses an episode
→ app downloads only that selected time fragment
→ app processes the clip for Reels
→ app adds subtitles
→ app generates/attaches a cover
→ final short video package is produced
```

## Core Product Principle

Do **not** download a full episode/video first when the user only needs a short fragment.

The first useful capability is:

```text
URL + start + end/duration → download only that selected fragment → save it as a clip
```

Local cutting of an already downloaded full video can be useful later, but it is **not** the first MVP path.


## MVP Chain

### MVP 1 — Manual Segment Download

Goal:

```text
Given a URL and a selected time range, download only that fragment and save it as a clip.
```

Input:

- source URL;
- start time;
- end time or duration;
- optional label;
- current quality/mode settings where practical.

Output:

- one saved clip file;
- metadata describing source URL, start, end, label, original title if known.

Implementation direction:

- Use yt-dlp section download capability where possible.
- The relevant yt-dlp concept is section/time-range download, for example using `--download-sections` with a `*START-END` range.
- Keep the implementation local and Windows-friendly.
- Save output under the configured output directory, for example:

```text
downloads/clips/<safe_source_title_or_id>/clip_001_<label>.mp4
```

Acceptance:

- A 30-second fragment from a long YouTube video can be downloaded without downloading the full video.
- Invalid ranges are rejected clearly.
- Output path remains constrained to the configured output directory.
- A real manual validation creates a playable clip file.

### MVP 2 — Transcript Episode Picker

Goal:

```text
Load transcript, show timestamped text, and allow selecting one transcript-based episode range.
```

Possible sources:

1. YouTube captions/transcript if available.
2. Existing `.srt`/`.txt` generated by local transcription.
3. Whisper transcription only as fallback when transcript is unavailable.

Output:

- selected episode start/end;
- selected transcript snippet;
- context window before/after;
- initial episode label/title.

### MVP 3 — AI Episode Finder

Goal:

```text
AI proposes semantic episodes from transcript timestamps.
```

AI output should be structured:

```json
{
  "episodes": [
    {
      "start": "00:12:10.000",
      "end": "00:12:42.000",
      "title": "Main conflict",
      "hook": "He explains why it failed",
      "reason": "Strong turn, emotion, and complete thought",
      "reels_angle": "conflict / insight / drama",
      "priority": 1
    }
  ]
}
```

### MVP 4 — Reels Processing

Goal:

```text
Convert downloaded episode into a vertical short-video asset.
```

Likely transformations:

- 9:16 output;
- 1080x1920 target;
- center crop or blurred background layout;
- audio normalization if needed;
- output metadata.

### MVP 5 — Subtitle Render

Goal:

```text
Use the accepted subtitle_studio / Remotion direction to render burned-in subtitles.
```

Use existing accepted Remotion proof as the donor implementation, but do not broaden it until segment download works.

### MVP 6 — Cover And Publishing Package

Goal:

```text
Create a complete package for a Reel/Short/TikTok.
```

Package example:

```text
clip_final.mp4
cover.jpg
captions.srt
transcript.txt
metadata.json
title.txt
description.txt
hashtags.txt
```

## Current Priority

The next executable task is **MVP 1 — Manual Segment Download**.

Do not start with:

- Chrome extension work;
- native-host E2E;
- full-video download then local cutting;
- AI automatic episode detection;
- timeline editor;
- Remotion changes;
- Video Content Analyzer;
